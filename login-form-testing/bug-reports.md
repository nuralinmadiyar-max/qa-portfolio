# Баг-репорты: Форма логина (saucedemo.com)

## Баг-репорт #1: Поле username/password принимает более 500 символов без ограничения

**Description:** Поля username и password на странице логина не имеют ограничения на максимальную длину ввода. Система принимает 500+ символов без валидации на стороне клиента.

**Шаги воспроизведения:**
1. Открыть страницу логина
2. Ввести в поле username строку из 500+ символов
3. Ввести в поле password строку из 500+ символов
4. Нажать Login

**Ожидаемый результат:** Поле username/password должно иметь разумное ограничение длины ввода (например, 50-100 символов) — либо визуально (maxlength), либо на уровне валидации.

**Фактический результат:** Максимальная длина полей username и password не ограничена. При вводе 500+ символов и нажатии Login выходит ошибка `Epic sadface: Username and password do not match any user in this service`.

**Severity:** Low — не ломает функциональность, является недоработкой (отсутствие ограничения ввода, а не сбой системы)

**Окружение:** Google Chrome 150.0.7871.101, Windows 11
